Tahap 1 — Jalankan di Lokal PC

Sebelum deploy ke internet, pastikan aplikasi berjalan dengan baik di komputer kamu terlebih dahulu. Ini penting untuk memastikan tidak ada error sebelum masuk ke server.

1.1 Software yang Harus Diinstall

SoftwareVersiLink DownloadKeterangan
Node.js22.x LTSnodejs.orgRuntime JavaScript untuk server
MySQL / XAMPP8.0+dev.mysql.com atau apachefriends.orgDatabase. XAMPP lebih mudah untuk pemula
Gitterbarugit-scm.comVersion control untuk upload kode
Tips: Jika menggunakan XAMPP, aktifkan modul MySQL saja (Apache tidak diperlukan). Akses phpMyAdmin di http://localhost/phpmyadmin.

1.2 Buat Database Lokal

Buka phpMyAdmin (http://localhost/phpmyadmin) atau MySQL Workbench, lalu jalankan perintah berikut:

CREATE DATABASE ptk_local CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
CREATE USER 'ptk_user'@'localhost' IDENTIFIED BY 'password123';
GRANT ALL ON ptk_local.* TO 'ptk_user'@'localhost';
FLUSH PRIVILEGES;
Ganti password123 dengan password yang kamu inginkan. Catat username dan password ini.

1.3 Jalankan Aplikasi

Buka Command Prompt atau PowerShell, masuk ke folder proyek:

cd c:\Users\DELL\Downloads\jobportalbm\ptk-app

Jalankan perintah berikut satu per satu:

npm install
npm run vendor:js
npm run build:assets

Set environment variables (di Command Prompt Windows):

set DATABASE_URL=mysql://ptk_user:password123@localhost/ptk_local
set SESSION_SECRET=ini-harus-minimal-32-karakter-acak-ya
set BASE_URL=http://localhost:3000
set NODE_ENV=development
set PORT=3000

Jalankan migrasi (membuat semua tabel database):

npm run migrate up

Jalankan server:

npm run dev

1.4 Verifikasi Lokal

Buka browser, akses: http://localhost:3000

Jika muncul halaman utama portal, Tahap 1 berhasil ✓

Cek juga health endpoint: http://localhost:3000/healthz — harus muncul {"status":"ok"}

Tahap 2 — Beli Hosting & Domain

2.1 Pilih Hosting yang Support Node.js

Penting: Tidak semua shared hosting support Node.js. Pastikan hosting yang kamu beli memiliki fitur "Setup Node.js App" di cPanel.
ProviderPaket MinimalHarga/bulanNode.jsRekomendasi
NiagahosterBusiness~Rp 50.000✅ cPanel + Node.js⭐⭐⭐⭐⭐
HostingerPremium~Rp 30.000✅ cPanel + Node.js⭐⭐⭐⭐

DomaiNesiaPersonal~Rp 35.000✅ cPanel + Node.js⭐⭐⭐⭐

2.2 Spesifikasi Minimum Hosting

  • Node.js versi 22 (atau minimal 18)
  • MySQL 8.0+ atau MariaDB 10.6+
  • Storage minimal 5 GB
  • RAM minimal 512 MB
  • cPanel dengan fitur "Setup Node.js App"

2.3 Beli Domain

Beli domain di provider yang sama dengan hosting (lebih mudah karena DNS sudah otomatis terhubung).

  • Domain .my.id — sekitar Rp 15.000/tahun (paling murah)
  • Domain .id — sekitar Rp 150.000/tahun
  • Domain .com — sekitar Rp 150.000/tahun

Contoh domain yang digunakan di proyek ini: buanamegahcareer.my.id

2.4 Setelah Beli Hosting

Kamu akan mendapat email berisi:

  • URL cPanel (biasanya: https://namadomain.com/cpanel atau https://server123.hosting.com:2083)
  • Username cPanel
  • Password cPanel
Simpan informasi login ini di tempat yang aman. Kamu akan sering membutuhkannya.

Tahap 3 — Setup Hosting (cPanel)

3.1 Buat Database MySQL di cPanel

  1. Login ke cPanel
  2. Cari dan klik MySQL Databases
  3. Di bagian "Create New Database", ketik nama database: ptk → klik Create Database
    (nama lengkap akan jadi: mycdmkay_ptk — prefix username otomatis ditambahkan)
  4. Di bagian "MySQL Users", buat user baru:
    Username: ptkuser   Password: buat password kuat (minimal 12 karakter, ada huruf besar, angka, simbol)
  5. Di bagian "Add User To Database", pilih user dan database yang baru dibuat → klik Add
  6. Centang ALL PRIVILEGES → klik Make Changes
Catat informasi ini:
Database name: mycdmkay_ptk
Username: mycdmkay_ptkuser
Password: (yang kamu buat)
Host: localhost

3.2 Upload Kode ke Hosting via GitHub

Cara terbaik adalah menggunakan Git. Ikuti langkah berikut:

A. Buat akun GitHub (jika belum punya)

Daftar di github.com — gratis.

B. Upload kode dari PC ke GitHub

Buka Command Prompt di folder proyek:

cd c:\Users\DELL\Downloads\jobportalbm\ptk-app
git init
git add .
git commit -m "initial commit"

Buat repository baru di GitHub (klik tombol "New" di github.com), lalu:

git remote add origin https://github.com/USERNAME/ptk-app.git
git branch -M main
git push -u origin main

C. Clone repository di cPanel

  1. Login cPanel → cari Git Version Control
  2. Klik Create
  3. Isi form:
    Clone URL: https://github.com/USERNAME/ptk-app.git
    Repository Path: /home/mycdmkay/ptk-app
  4. Klik Create

3.3 Setup Node.js App di cPanel

  1. cPanel → cari Setup Node.js App
  2. Klik Create Application
  3. Isi form sesuai tabel berikut:
FieldValue
Node.js version22
Application modeProduction
Application root/home/mycdmkay/ptk-app
Application URLbuanamegahcareer.my.id
Application startup fileartifacts/api-server/dist/index.mjs
  1. Klik Create

3.4 Set Environment Variables

Masih di halaman Setup Node.js App, scroll ke bawah ke bagian Environment Variables. Tambahkan variabel berikut satu per satu dengan klik Add Variable:

Variable NameValue / Keterangan
DATABASE_URLmysql://mycdmkay_ptkuser:PASSWORD@localhost/mycdmkay_ptk

SMTP_PORT587

SMTP_PASSApp Password Gmail (bukan password biasa)

Cara generate SESSION_SECRET:
Di cPanel → Terminal, ketik: openssl rand -hex 32
Copy hasilnya (64 karakter hex) dan paste sebagai value SESSION_SECRET.
Cara buat App Password Gmail:
1. Login Gmail → Kelola Akun Google → Keamanan
2. Aktifkan Verifikasi 2 Langkah (jika belum)
3. Cari "Sandi Aplikasi" → buat sandi baru untuk "Mail"
4. Copy 16 karakter yang muncul → paste sebagai SMTP_PASS

Tahap 4 — Install & Build di Server

Buka Terminal di cPanel (cari menu "Terminal" di cPanel). Jalankan perintah berikut satu per satu:

4.1 Masuk ke Folder Aplikasi

cd ~/ptk-app

4.2 Install Dependencies

npm ci --omit=dev

Proses ini mengunduh semua library yang dibutuhkan. Tunggu sampai selesai (bisa 2-5 menit).

4.3 Download Library Frontend

npm run vendor:js

Mengunduh htmx, Alpine.js, dan Sortable.js.

4.4 Build Aplikasi

npm run build

Mengkompilasi TypeScript menjadi JavaScript. Hasil ada di folder artifacts/api-server/dist/.

4.5 Jalankan Migrasi Database

node tools/migrate.mjs up

Membuat semua tabel di database. Harus berhasil tanpa error.

Verifikasi: Buka phpMyAdmin di cPanel → pilih database mycdmkay_ptk → pastikan ada banyak tabel (users, sessions, job_postings, applications, dll).

4.6 Buat Folder Log

mkdir -p ~/logs

4.7 Restart Aplikasi

mkdir -p ~/ptk-app/tmp && touch ~/ptk-app/tmp/restart.txt

Atau klik tombol Restart di halaman Setup Node.js App.

Tahap 5 — Setup Cron Jobs

Cron jobs adalah tugas terjadwal yang berjalan otomatis di background (kirim email, backup, dll).

cPanel → Cron Jobs → tambahkan 7 entri berikut. Untuk setiap entri, isi kolom "Command" dengan perintah di bawah:

Ganti mycdmkay dengan username cPanel kamu yang sebenarnya di setiap perintah.

Cron 1 — Mail Flush (setiap 2 menit)

Minute: */2   Hour: *   Day: *   Month: *   Weekday: *

/home/mycdmkay/nodevenv/ptk-app/22/bin/node /home/mycdmkay/ptk-app/artifacts/api-server/dist/crons/index.mjs mail-flush >> /home/mycdmkay/logs/cron-mail.log 2>&1

Tahap 6 — Aktifkan SSL & Restart

6.1 Aktifkan SSL (HTTPS)

  1. Login cPanel → cari SSL/TLS Status
  2. Klik tombol Run AutoSSL
  3. Tunggu beberapa menit sampai domain kamu menampilkan ikon gembok hijau ✅
SSL gratis dari Let's Encrypt akan otomatis dipasang. Ini membuat website kamu bisa diakses dengan https:// yang aman.

6.2 Restart Aplikasi

Di Terminal cPanel:

touch ~/ptk-app/tmp/restart.txt

Atau di halaman Setup Node.js App → klik tombol Restart.

6.3 Pasang .htaccess

Di Terminal cPanel:

cd ~/ptk-app && npm run deploy:htaccess

Ini menyalin konfigurasi web server ke ~/public_html/.htaccess untuk redirect HTTP → HTTPS dan optimasi performa.

Troubleshooting — Masalah Umum

Error / GejalaPenyebabSolusi
Health check mengembalikan 503 DATABASE_URL salah atau database belum dibuat Cek nama database, username, password di env vars. Pastikan migrasi sudah dijalankan.
Halaman tidak muncul / 502 Bad Gateway Startup file salah atau build belum dijalankan Pastikan npm run build berhasil. Cek startup file: artifacts/api-server/dist/index.mjs
npm run build gagal Node.js versi salah atau dependency belum install Pastikan Node.js 22 aktif. Jalankan npm ci --omit=dev terlebih dahulu.
Email tidak terkirim SMTP settings salah Cek SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS. Untuk Gmail, gunakan App Password.
SSL tidak aktif / masih HTTP AutoSSL belum selesai atau domain belum propagasi Tunggu 24 jam untuk propagasi DNS. Jalankan ulang AutoSSL di cPanel.
Cron tidak berjalan Path Node.js salah Cek path Node.js: di Terminal ketik which node atau cek di Setup Node.js App.
Error "Missing required environment variable" Env var belum diset di cPanel Buka Setup Node.js App → Environment Variables → tambahkan yang kurang.

Cara Cek Log Error

Di Terminal cPanel:

# Log aplikasi utama
tail -100 ~/logs/error.log

# Log cron mail
tail -50 ~/logs/cron-mail.log

# Log cron backup
tail -50 ~/logs/cron-backup.log

Di cPanel → Errors (menu di bagian Metrics) untuk melihat error Apache/Passenger.

Prosedur Update Kode

Setiap kali ada perubahan kode, ikuti langkah berikut:

Di PC Lokal (setelah edit kode)

cd c:\Users\DELL\Downloads\jobportalbm\ptk-app
git add .
git commit -m "deskripsi perubahan"
git push origin main

Di Server (cPanel Terminal)

cd ~/ptk-app

# Pull perubahan terbaru
git pull

# Install dependency baru (jika ada)
npm ci --omit=dev

# Build ulang
npm run build

# Jalankan migrasi baru (jika ada)
node tools/migrate.mjs up

# Restart aplikasi
touch tmp/restart.txt
Tips: Selalu jalankan node tools/migrate.mjs up setelah update kode, karena mungkin ada perubahan struktur database. Perintah ini aman dijalankan berulang kali — hanya menjalankan migrasi yang belum pernah dijalankan.
